# Analyze / Portfolio / 持股診斷視角 對照與驗收標準

> 類型：產品對照 / 驗收基準文件
> 建立日期：2026-03-16
> 用途：作為後續需求拆解與驗收時的對照表，避免新倉分析、持股列表、持股診斷三條路線再次混淆

---

## 1. 目的

本文件用來回答三個問題：

1. 哪些優化主要影響 Analyze 頁
2. 哪些優化應落在持股列表頁（Portfolio）
3. 哪些語義必須保留在持股診斷視角，不能被 Analyze 頁混用

這份文件不取代實作計劃，而是作為後續每張需求文件的驗收母表。

---

## 2. 頁面角色總覽

| 介面 / 視角  | 主要角色            | 核心問題                                   | 不可混淆的語義                           |
| ------------ | ------------------- | ------------------------------------------ | ---------------------------------------- |
| Analyze 頁   | 新倉策略建議        | 這檔股票現在是否值得觀察、等待或建立新倉   | 不是持股中的續抱 / 減碼 / 出場指令       |
| Portfolio 頁 | 持股列表 / 決策入口 | 我目前有哪些持股、哪幾檔需要優先關注       | 不是單檔完整診斷頁，也不是新倉策略主畫面 |
| 持股診斷視角 | 持股操作建議        | 我已持有這檔股票，現在該續抱、減碼還是出場 | 只處理持股上下文，不負責新倉評估         |

## 3. 高層對照

| 主題                   | Analyze 頁             | Portfolio 頁       | 持股診斷視角                 |

## 4. Analyze 頁驗收標準

**頁面定位：** 新倉策略建議頁

### 4.1 必須維持的語義

1. 頁面底部策略區塊必須明確屬於「新倉策略建議」
2. 不得把 Analyze 頁的策略卡描述成持股中的即時操作指令
3. Analyze 頁可提供觀察、等待、分批進場等建議，但不直接承擔續抱 / 減碼 / 出場判斷

### 4.2 後續優化後應新增的可視資訊

若新倉策略演算法升級完成，Analyze 頁應逐步承接以下資訊：

1. `conviction_level`
2. `thesis_points`
3. `upgrade_triggers`
4. `downgrade_triggers`
5. `invalidation_conditions`
6. `suggested_position_size`
7. 盤中 / 收盤狀態下的策略積極度差異提示

### 4.3 建議前端呈現形態

優先改為四段式：

1. 建議動作
2. 主要理由
3. 關鍵價位
4. 失效條件

### 4.4 驗收條件

1. 使用者能在頁面上看懂這是新倉視角，而不是持股視角
2. 使用者能看懂建議背後的主要理由
3. 使用者能看懂什麼條件下原策略會失效
4. 在低信心或盤中情境下，頁面會清楚顯示保守化訊號，而不是只給出積極建議

---

## 5. Portfolio 頁驗收標準

**頁面定位：** 持股列表與決策入口頁

### 5.1 必須維持的語義

1. Portfolio 頁的主要任務是管理與瀏覽持股列表
2. 它應成為決策入口，但不是取代 Analyze 或 Position 的完整分析頁
3. 不應把完整的新倉策略卡直接塞進列表卡片，造成語義混亂與資訊過載

### 5.2 後續優化後可新增的可視資訊

Portfolio 頁較適合新增摘要型資訊，而非完整分析內容：

1. 最近一次持股診斷結果摘要
2. 最近一次分析時間
3. 目前判斷狀態是盤中版或收盤版
4. 是否存在出場警示
5. 是否存在需要優先關注的持股
6. 從持股列表切換到 Analyze 視角或 Position 視角的入口

### 5.3 後續優化後可新增的操作功能

1. 從某檔持股快速進入新倉分析視角
2. 從某檔持股快速進入持股診斷視角
3. 在列表層顯示「需關注」或「有警示」的優先排序

### 5.4 驗收條件

1. 使用者能從列表快速知道哪幾檔需要先看
2. 使用者能從列表快速跳進對應的決策頁
3. 列表提供的是摘要與入口，不是把完整分析頁硬塞進表格
4. 新倉與持股兩種視角若同時存在，必須清楚分流，不得混成同一種建議語言

---

## 6. 持股診斷視角驗收標準

**介面定位：** 持股操作建議視角（目前由 Portfolio 頁內診斷 modal 承接）

### 6.1 必須維持的語義

1. 持股診斷視角只處理已持有股票的決策
2. `recommended_action` 維持 `續抱 / 減碼 / 出場` 語義
3. `exit_reason` 仍屬持股風險或出場判斷的重要依據
4. 不應把新倉策略欄位直接混進持股診斷視角的主要決策邏輯

### 6.2 後續可受益但不應改變定位的項目

1. 盤中 / 收盤狀態提示更清楚
2. 分析時間與資料品質提示更清楚
3. 若未來做版本化，可保留診斷版本資訊

### 6.3 不建議直接塞入的內容

1. 新倉策略的 `thesis_points`
2. 新倉策略的 `suggested_position_size`
3. 把持股診斷視角改成「重新評估是否值得買」視角

### 6.4 驗收條件

1. 使用者能清楚知道這是持股中的操作建議
2. 續抱 / 減碼 / 出場語義不被 Analyze 頁的新倉策略稀釋
3. 持股診斷視角若新增輔助資訊，也不能破壞持股決策的主軸

## 7. 後續優化主題對應頁面

| 優化主題             | 主要頁面            | 次要頁面                 | 驗收焦點                           |
| -------------------- | ------------------- | ------------------------ | ---------------------------------- |
| 新倉策略回測與校準   | Analyze             | Portfolio                | 策略是否更可靠，而非畫面是否變多   |
| 信心分數校準         | Analyze             | Portfolio / 持股診斷視角 | 高分是否真的更可信                 |
| Analyze 頁策略卡升級 | Analyze             | 無                       | 是否更可讀、更可解釋               |
| 盤中 vs 收盤策略分流 | Analyze             | Portfolio / 持股診斷視角 | 是否正確保守化，不誤導使用者       |
| 資料源品質監控       | Portfolio / Analyze | 持股診斷視角             | 是否能看出資料問題而非誤判策略問題 |
| LLM 輸出評測         | Analyze             | 持股診斷視角             | 分析文字是否穩定、不越界           |
| 策略版本化           | Analyze / 後台      | Portfolio                | 是否可追蹤策略邏輯版本             |
| 新倉 / 持股體驗整合  | Portfolio           | Analyze / 持股診斷視角   | 是否做出雙視角而不混語意           |

---

## 8. 驗收使用方式

後續每次新增需求文件時，建議都回頭對照這份文件，至少回答以下問題：

1. 這次主要影響 Analyze、Portfolio，還是持股診斷視角？
2. 這次新增的是完整分析、摘要資訊，還是入口操作？
3. 這次有沒有把新倉與持股語義混在一起？
4. 這次的前端變更是否能對應到後端新增的可靠資訊，而不是只增加視覺包裝？

---

## 9. 結論

你後續若圍繞新倉策略路線繼續優化，最先被強化的一定是 Analyze 頁；但持股列表頁並不是不更新，而是它的角色會更偏向「監控與分流入口」。

因此後續驗收時，建議用下面這句話快速檢查：

1. Analyze 頁看的是新倉
2. Portfolio 頁看的是持股清單與決策入口
3. 持股診斷視角看的是持股中的當前操作

只要這三句還成立，代表語義邊界大致沒有被破壞。

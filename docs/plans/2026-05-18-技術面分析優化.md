## 執行原則

- 第一階段先做分析內容可用，不先碰前端版面與過多策略邏輯。
- 優先把指標計算收斂到共用數學層，避免在多個節點重算出不同結果。
- 若新指標要進入 rule-based 判斷，必須明確定義加分、扣分、衝突與降級規則。
- 歷史快取若要支援昨日連續性分析，不能只補 5 日資料，至少需要 3 個月日線。
- 每完成一層就補對應測試，不接受只改 prompt 不改測試。

### Task 2：技術敘事加入布林通道與 MACD（已完成）

修改 context_generator.py

工作內容：

- 新增布林通道敘事函式
- 新增 MACD 敘事函式
- 在 generate_technical_context 中把兩段敘事接入 technical_context

布林通道敘事建議方向：

- 價格位於上軌附近：短線偏熱，追價風險提高
- 價格位於中軌上方且未碰上軌：趨勢偏強但未極端
- 價格接近下軌：偏弱或超跌，留意是否止跌
- 通道擴張：波動放大
- 通道收斂：波動壓縮，留意後續表態

MACD 敘事建議方向：

- MACD 線在訊號線上方且柱狀體擴大：多方動能增強
- MACD 線上穿訊號線：黃金交叉
- MACD 線下穿訊號線：死亡交叉
- 位於零軸上方：中期偏多
- 位於零軸下方：中期偏弱
- 柱狀體收斂：原趨勢動能減弱

完成定義：

- technical_context 固定含布林通道與 MACD 段落
- 敘事是 rule-based，可重現，不依賴 LLM
- 資料不足時有明確 fallback 文案

交付結果：`generate_technical_context()` 已固定輸出 `【布林通道】` 與 `【MACD】` 段落，並針對資料不足、靠近上軌 / 下軌、波動收斂 / 擴張、零軸位階與多空動能提供固定文案。

## 風險與決策點

### 風險 1：只改 prompt，不改 rule-based 層

結果會是 tech_insight 文字看起來變完整，但 technical_signal、confidence_score、strategy 還是舊邏輯，造成前後不一致。

決策：

- 最低限度也要完成 Phase 1 全套，不能只改 prompt。

### 風險 2：昨日回填仍維持 5d

結果會是昨日指標不可靠，MACD 幾乎無法正確計算，連續性分析失真。

決策：

- history backfill 必須升到 3mo，這是必要改動，不是優化項。

### 風險 3：新欄位命名鬆散

結果會出現 macd、macd_line、macd_value 等同義欄位混用，後續維護成本上升。

決策：

- 第一階段先定名，再擴散到所有模組。

### 風險 4：過度擴大範圍

若一開始就同時改前端、API、策略、score，容易讓改動面過大。

決策：

- 先做 Phase 1。
- Phase 1 驗證穩定後，再做 Phase 2。
- Phase 3 只有在產品明確需要顯示數值時才啟動。
